# 实验报告

## 一、实验题目

**面向迷宫竞赛机器人的路径规划优化研究**

---

## 二、问题背景

在世界迷宫大赛、校园智能机器人竞赛以及相关智能车竞赛中，机器人需要在有限时间内从起点出发，在复杂迷宫环境中快速、稳定地到达目标区域。表面上看，这一问题是“走出迷宫”，但从运筹学角度看，本质上是一个**带约束的路径优化决策问题**。

在实际竞赛场景中，机器人不仅要“找到路”，还要尽可能满足以下要求：

1. 路径长度尽可能短；
2. 搜索过程尽可能高效；
3. 转弯次数尽可能少，以降低控制难度；
4. 尽量避开风险区域或狭窄区域；
5. 当环境发生变化时，能够快速重新规划路径。

因此，本项目将迷宫问题统一建模为一个静态与动态结合的路径规划问题，并基于同一批竞赛迷宫数据，对多种经典与改进算法进行对比分析。

本项目使用的地图主要来源于历年 Micromouse 竞赛公开迷宫，并统一转换为 `16 x 16` 网格地图格式，以保证实验数据真实、可复现、可比较。

---

## 三、研究目标

本实验的主要目标如下：

1. 建立统一的迷宫路径规划数学模型；
2. 在统一地图和统一接口下实现多种路径规划算法；
3. 比较静态算法与动态算法在路径质量、搜索效率和环境适应性上的差异；
4. 构建可视化展示体系，以支持结果分析与答辩展示；
5. 在课程项目层面形成一条清晰的算法递进主线：

\[
\text{能找到路} \rightarrow \text{更快找到路} \rightarrow \text{找到更优的路} \rightarrow \text{环境变化后快速改路}
\]

---

## 四、数学建模

### 4.1 地图建模

将迷宫环境离散化为二维网格图：

\[
G=(V,E)
\]

其中：

1. \(V\) 表示可通行节点集合；
2. \(E\) 表示相邻节点之间的可通行边；
3. 起点记为 \(S\)；
4. 目标区域记为 \(G_o\)。

每个网格单元通过 `wall_matrix` 记录四个方向是否存在墙体，从而构成标准的图结构。对于 Micromouse 标准迷宫，终点通常为中心 `2 x 2` 区域，因此项目中同时保留 `goal` 与 `goal_cells` 两种表示方式。

---

### 4.2 静态最短路径模型

在最基础的静态无权模型中，目标是从起点到终点求一条最短路径：

\[
\min L(P)
\]

其中：

1. \(P\) 表示从起点到终点的可行路径；
2. \(L(P)\) 表示路径长度或步数。

这一模型适用于 BFS、DFS 和基础 A* 的基线分析，其中 BFS 可作为最短步数的参考标准，DFS 则作为深度优先搜索的补充对照。

---

### 4.3 综合代价模型

为了更贴近机器人竞赛中的实际决策需求，本项目在最短路模型基础上进一步引入转弯代价、风险区域代价和狭窄区域代价，构造综合代价函数：

\[
\min C(P)=p_1 L(P)+p_2 T(P)+p_3 R(P)+p_4 Q(P)
\]

其中：

1. \(L(P)\)：路径长度；
2. \(T(P)\)：转弯代价；
3. \(R(P)\)：风险区域代价；
4. \(Q(P)\)：狭窄区域代价；
5. \(p_1,p_2,p_3,p_4\)：对应代价权重。

在当前项目中，推荐参考权重为：

1. \(p_1=1.0\)
2. \(p_2=1.0\)
3. \(p_3=3.0\)
4. \(p_4=2.0\)

该模型主要用于 `Cost-aware A*` 与 `Weighted A*`。

---

### 4.4 方向状态扩展模型

为了显式描述转弯惩罚，本项目在改进 A* 中采用方向状态扩展，将单纯的节点位置 \(v\) 扩展为状态：

\[
(v,d)
\]

其中：

1. \(v\) 表示当前位置；
2. \(d\) 表示进入该位置时的方向。

这样可以自然地区分：

1. 直行；
2. 90 度转弯；
3. 180 度掉头。

在项目中，统一采用如下转弯计数规则：

\[
turn\_count = N_{90} + 2N_{180}
\]

该模型使路径优化从“只看经过哪些格子”升级为“看机器人以什么方式经过这些格子”，更符合竞赛机器人控制特点。

---

### 4.5 动态环境建模

在动态场景下，地图会随着时间更新。项目中使用 `dynamic_updates` 描述环境变化：

```json
{
  "step": 20,
  "blocked_cells": [[5, 6], [5, 7]],
  "released_cells": []
}
```

也就是说，在第 `step` 步时，若干新障碍会出现或解除。此时路径规划问题变为：

\[
P_{t+1}=\arg\min C_{t+1}(P)
\]

其中 \(C_{t+1}(P)\) 为地图更新后的综合代价函数。该模型对应 `LPA*` 与 `D* Lite` 的动态重规划问题。

---

## 五、算法说明

### 5.1 BFS

`BFS` 采用广度优先搜索，在无权静态图中可以保证找到最短步数路径。其特点是：

1. 能稳定找到最短可达路径；
2. 搜索范围往往较大；
3. 适合作为静态无权基准算法。

在本项目中，`BFS` 的主要作用是提供最短步数参考值，并为后续算法效率对比提供基础。

---

### 5.2 DFS

`DFS` 采用深度优先搜索。它的特点是：

1. 实现简单；
2. 搜索更偏向“一条路走到底”；
3. 不保证最短路径；
4. 更适合作为补充对照，而不是最优解算法。

在本项目中，`DFS` 用来说明“找到一条可行路径”与“找到最优路径”并不是同一件事。

---

### 5.3 A*

`A*` 的评价函数为：

\[
f(n)=g(n)+h(n)
\]

其中：

1. \(g(n)\) 表示从起点到当前节点的实际代价；
2. \(h(n)\) 表示从当前节点到目标的启发式估计代价。

在网格迷宫中，启发函数通常采用曼哈顿距离：

\[
h(n)=|x_n-x_g|+|y_n-y_g|
\]

`A*` 的目标是在仍保持较优路径质量的前提下，减少节点扩展数量，提高搜索效率。

---

### 5.4 Cost-aware A*

`Cost-aware A*` 是在 `A*` 的基础上进一步引入综合代价模型，并采用方向状态扩展。它的特点是：

1. 不再只优化路径长度；
2. 同时考虑转弯代价、风险区代价和狭窄区代价；
3. 通过方向状态建模显式刻画转弯惩罚。

其本质上仍采用：

\[
f(n)=g(n)+h(n)
\]

但其中的 \(g(n)\) 已经不再是简单步数，而是综合代价累计值。

---

### 5.5 Weighted A*

`Weighted A*` 在 `Cost-aware A*` 的基础上进一步引入启发函数权重：

\[
f(n)=g(n)+w\cdot h(n),\quad w>1
\]

其中：

1. \(g(n)\) 仍为综合代价累计值；
2. \(h(n)\) 为最小曼哈顿距离估计；
3. \(w\) 为启发函数权重，项目中默认采用 `1.5`。

其特点是：

1. 搜索更偏向目标方向；
2. 通常扩展节点更少或路径更激进；
3. 可能在路径最优性和搜索速度之间形成新的权衡。

因此，`Cost-aware A*` 与 `Weighted A*` 的差异主要在于：

1. 前者强调“综合代价模型”；
2. 后者强调“启发函数加权后的搜索策略”。

---

### 5.6 LPA*

`LPA*` 是一种增量式最短路算法，适用于图结构发生局部变化的场景。其核心思想是：

1. 保存已有搜索结果；
2. 地图更新后只对受影响节点进行修补；
3. 避免完全从头搜索。

它更偏理论化，更适合展示动态最短路中的增量更新思想。

---

### 5.7 D* Lite

`D* Lite` 也是一种动态环境下的增量重规划算法，更适合移动机器人在线导航场景。与 `LPA*` 相比，它更强调：

1. 机器人当前位置的不断变化；
2. 障碍出现后围绕当前状态做局部修补；
3. 在线运行中的快速改路能力。

在本项目中，`D* Lite` 与 `LPA*` 使用同一批动态事件进行测试，以比较二者在动态环境下的适应性。

---

## 六、算法比较与评判标准

为保证不同算法的结果可直接比较，本项目采用统一评判标准。

### 6.1 评判指标

所有算法统一记录以下指标：

1. `success`：是否成功找到路径；
2. `path_length`：路径长度；
3. `turn_count`：转弯次数；
4. `risk_cost`：风险区域代价；
5. `narrow_cost`：狭窄区域代价；
6. `explored_nodes`：搜索扩展节点数；
7. `runtime_ms`：运行时间；
8. `total_cost`：综合总代价。

动态算法额外记录：

1. `replan_count`：重规划次数；
2. `replan_time_ms`：平均重规划时间；
3. `updated_nodes`：受影响节点数；
4. `replanned_path_length`：更新后路径长度。

---

### 6.2 时间统计方法

为保证结果可比，统一采用如下计时原则：

1. 只统计算法核心求解时间；
2. 不计入地图读取、打印、保存和绘图时间；
3. 动态算法的总时间为首次规划时间与所有重规划时间之和；
4. 若做正式实验，应对每张地图每种算法重复运行 `10` 次，去掉最大值和最小值后取平均。

---

### 6.3 综合代价计算方式

项目中统一采用：

\[
total\_cost = p_1\cdot path\_length + p_2\cdot turn\_count + p_3\cdot risk\_cost + p_4\cdot narrow\_cost + p_5\cdot replan\_count
\]

当前推荐参考值为：

1. \(p_1=1.0\)
2. \(p_2=1.0\)
3. \(p_3=3.0\)
4. \(p_4=2.0\)
5. \(p_5=2.0\)

对于静态算法，一般有 `replan_count = 0`。

## 七、实验环境配置

### 7.1 硬件与系统环境

项目当前默认在个人电脑环境下运行，推荐环境如下：

1. Python 3.9 及以上；
2. 无需额外第三方图形库即可运行项目主体。

---

### 7.2 项目结构

核心目录如下：

```text
opt_final/
├── maps/
├── scripts/
├── src/
├── outputs/
├── README.md
├── 接口规范.md
├── 计划书.md
└── 实验报告.md
```

其中：

1. `maps/`：原始迷宫与 JSON 地图；
2. `scripts/`：地图转换、读取等脚本；
3. `src/`：主流程与算法实现；
4. `outputs/`：实验结果输出目录。

---

### 7.3 输出结构

主流程一次运行后，输出统一写入：

```text
outputs/<run_name>/
├── results/
│   ├── run_result.json
│   └── run_summary.json
└── dynamic_maps/
```

可视化单独汇总到：

```text
outputs/visualizations/<run_name>/
```

其中每个算法会输出：

1. `PNG` 过程图；
2. `GIF` 动态图。

## 八、如何使用

### 8.1 地图转换

将原始 Micromouse 文本地图转换为统一 JSON 格式：

```bash
python3 'scripts/convert_micromouse_maps.py'
```

---

### 8.2 读取地图

检查地图是否能正常载入：

```bash
python3 'scripts/load_map.py'
```

---

### 8.3 运行主流程

运行一整套主流程并生成结果：

```bash
python3 'src/main.py' --output-dir 'outputs/demo_run' --visualize-mode image
```

---

### 8.4 只运行指定算法

如果只想运行部分算法，可以使用 `--algorithms` 参数：

```bash
python3 'src/main.py' \
  --output-dir 'outputs/demo_run' \
  --algorithms 'bfs,a_star,weighted_a_star' \
  --visualize-mode image
```

目前支持的算法键包括：

1. `bfs`
2. `dfs`
3. `a_star`
4. `cost_aware_a_star`
5. `weighted_a_star`
6. `lpa`
7. `dstar_lite`

---

### 8.5 运行综合测试

运行覆盖全项目算法的 smoke test：

```bash
python3 'test_comprehensive.py'
```

输出结果会保存在：

```text
outputs/tests/comprehensive_results.json
```

---

## 九、实验结果分析建议

在正式撰写最终课程论文或汇报时，可按以下思路分析实验结果：

1. `BFS` 与 `DFS`
   用于说明“可达”与“最优”不是同一个问题；
2. `BFS` 与 `A*`
   用于说明启发式搜索在效率上的优势；
3. `A*` 与 `Cost-aware A*`
   用于说明从“最短路径”到“综合代价最优路径”的升级；
4. `Cost-aware A*` 与 `Weighted A*`
   用于说明启发函数加权如何改变搜索行为；
5. `LPA*` 与 `D* Lite`
   用于分析动态环境下增量更新与在线重规划的差异。

同时建议结合项目生成的 `PNG` 和 `GIF` 可视化结果进行展示，以增强实验分析的直观性。

## 十、结论

本实验围绕迷宫竞赛机器人路径规划问题，构建了一条从静态最短路到动态重规划的完整算法主线，并在统一地图、统一接口和统一评判标准下实现了多算法比较。项目当前已经完成：

1. 地图数据统一与格式转换；
2. 静态算法 `BFS`、`DFS`、`A*`、`Cost-aware A*`、`Weighted A*`；
3. 动态算法 `LPA*` 与 `D* Lite`；
4. `PNG + GIF` 双形式可视化；
5. 主流程调度、结果输出与综合测试。

从课程项目角度看，本实验较完整地展示了路径规划问题在运筹学与智能搜索算法中的建模、优化与比较过程，具有较好的展示性、可扩展性和答辩表现力。
