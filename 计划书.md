# 计划书

## 一、课题名称

**面向迷宫竞赛机器人的路径规划优化研究**

## 二、问题背景与情境包装

相比“抽象迷宫求最短路”，我们希望将问题包装在一个更具体、也更贴近大学生真实认知与竞赛语境的场景中。

在世界迷宫大赛、校园智能机器人竞赛或相关创新实践项目中，参赛机器人需要在限定时间内从迷宫起点出发，穿越复杂障碍区域，尽快到达目标位置。比赛中的核心任务表面上是“走出迷宫”，但从运筹学角度看，本质上是一个**带约束的路径优化决策问题**。

机器人在比赛过程中通常会面临以下现实约束：

1. 迷宫赛道中存在窄通道、急转角、死胡同和复杂障碍布局；
2. 目标不仅是“能到达终点”，还要尽可能缩短路径、减少耗时；
3. 频繁转弯会增加控制难度，也可能降低机器人运行稳定性；
4. 某些区域可能更狭窄、更拥堵或更靠近障碍边界，通行风险更高；
5. 比赛过程中可能出现临时障碍、局部封路或环境信息更新，需要重新规划路径。

因此，本项目不把迷宫看作单纯的“益智问题”，而是将其抽象为竞赛环境下的路径规划优化问题：在已知或逐步更新的地图中，寻找一条从起点到终点的最优路径，使其综合兼顾路径长度、转弯代价、风险代价、搜索效率以及动态重规划能力。

这样的包装有两个优点：

1. 保留了迷宫场景直观、易展示的特点；
2. 同时增强了课题的现实感、竞赛感和运筹学优化色彩。

---

## 三、研究目标

本项目希望围绕同一个竞赛场景，构建一条清晰的算法优化主线，而不是简单罗列若干经典算法。具体目标如下：

1. 将迷宫竞赛环境抽象为图模型，建立路径规划数学模型；
2. 从静态无权最短路出发，逐步过渡到启发式搜索与多目标优化；
3. 通过方向状态扩展，将转弯惩罚正式纳入路径代价模型；
4. 统一地图、算法输入输出接口与实验评判标准，保证四位成员的结果可直接比较；
5. 比较不同算法在路径质量、搜索效率和动态适应性方面的表现；
6. 在环境发生变化时实现快速重规划，体现算法优化的递进关系。

---

## 四、问题建模

### 4.1 环境建模

将迷宫竞赛场地离散化为二维网格图，记为：

\[
G=(V,E)
\]

其中：

* \(V\) 表示可通行位置集合；
* \(E\) 表示相邻可通行位置之间的连接边；
* 起点 \(S\) 表示机器人出发区域；
* 终点 \(G_o\) 表示比赛目标区域。

在基础模型中，若所有可行边代价相同，则问题可转化为无权图最短路问题。

从几何意义上看，迷宫中的墙体、障碍、窄通道和转角均可通过网格地图直观表示，因此该建模方式既便于理论分析，也便于程序实现和实验展示。

为便于统一实验，建议默认采用 Micromouse 竞赛中常见的标准迷宫规格，即以 `16 x 16` 网格作为主要测试尺度；若后续需要补充复杂场景，可再增加更大尺寸或人工扰动版本，但所有算法都应基于同一批标准地图进行比较。

---

### 4.2 基础优化目标：最短可达路径

在静态、无权、地图完全已知的条件下，目标是求一条从起点到终点的最短路径：

\[
\min L(P)
\]

其中：

* \(P\) 为从 \(S\) 到 \(G_o\) 的可行路径；
* \(L(P)\) 为路径长度或步数。

这一阶段主要回答的问题是：

**机器人能否找到一条最短可达路径。**

---

### 4.3 扩展优化目标：多因素综合代价

在实际竞赛中，仅追求最短步数并不一定最优。为了体现更明显的运筹学优化特征，我们将目标扩展为综合代价最小：

\[
\min C(P)=\alpha L(P)+\beta T(P)+\gamma R(P)+\delta Q(P)
\]

其中：

* \(L(P)\) 表示路径长度；
* \(T(P)\) 表示转弯代价；
* \(R(P)\) 表示靠近障碍或高风险区域时的通行代价；
* \(Q(P)\) 表示狭窄区域或拥堵区域的附加代价；
* \(\alpha,\beta,\gamma,\delta\) 为权重系数。

该模型体现了从“单目标最短路”向“多因素带权路径优化”的升级，也使得课题更符合运筹学中多目标、多约束决策问题的特点。

---

### 4.4 方向状态扩展建模

若只把“位置”作为节点，就很难严格刻画转弯惩罚。因此可将状态进一步扩展为：

\[
(v,d)
\]

其中：

* \(v\) 表示当前位置；
* \(d\) 表示进入该位置时的运动方向。

这样一来：

* 直行对应较低转移代价；
* 转弯对应额外惩罚；
* 路径优化不再只是“走到哪个格子”，而是“以什么朝向走到该格子”。

这一建模方式能够显著增强模型的理论深度，也是本项目最能体现“优化升级”的关键点之一。

---

### 4.5 动态环境建模

比赛过程中，机器人可能在运行中发现新的障碍或赛道局部封闭，导致地图从 \(G_t\) 更新为 \(G_{t+1}\)。此时原路径可能失效，需要重新规划：

\[
P_{t+1}=\arg\min C_{t+1}(P)
\]

因此，本项目不仅研究静态最优路径，也研究**动态环境更新下的快速重规划问题**。

---

## 五、算法设计与优化递进关系

本项目不再采用“几个算法并列比较”的表达方式，而是采用“同一场景下问题逐步复杂化、算法逐步优化”的主线。整体递进关系如下：

**第 1 层：静态无权最短路**
BFS 求最短路径，建立基准结果。

**第 2 层：在同一目标下优化搜索效率**
A* 利用启发函数减少节点扩展数，提高求解速度。

**第 3 层：从最短距离升级为综合代价最小**
方向状态扩展的带权 A* 处理转弯惩罚、风险代价与狭窄区域代价。

**第 4 层：考虑环境变化后的快速重规划**
LPA* 与 D* Lite 处理地图更新场景，实现增量式重规划。

也就是说，项目主线不是“会几种算法”，而是：

\[
\text{能找到路} \rightarrow \text{更快找到路} \rightarrow \text{找到更优的路} \rightarrow \text{环境变化后快速改路}
\]

这条主线更能体现算法优化的递进关系。

---

### 5.1 第一层：BFS 作为基准算法

在完全已知、无权、静态地图中，BFS 能保证求得最短步数路径，因此适合作为全项目的基准算法。

这一层的意义在于：

1. 给出所有实验的基础最短路径长度；
2. 为后续算法的效率比较提供参照；
3. 说明在最简单模型下，问题可以由经典最短路方法解决。

这一层重点回答：

**能不能稳定找到最短可达路径。**

说明：DFS 可以保留为补充对照，只用于说明“可达不等于最优”，不建议作为核心成果之一。

---

### 5.2 第二层：A* 的搜索效率优化

在目标仍然是“最短路径”的前提下，我们进一步优化搜索过程本身。

A* 的评价函数为：

\[
f(n)=g(n)+h(n)
\]

其中：

* \(g(n)\) 表示从起点到当前节点的实际代价；
* \(h(n)\) 表示从当前节点到终点的启发式估计代价。

在网格迷宫环境中，\(h(n)\) 可取曼哈顿距离：

\[
h(n)=|x_n-x_g|+|y_n-y_g|
\]

这一层的核心不是改变优化目标，而是在**仍求最短路的前提下减少搜索节点数和运行时间**，体现“效率优化”。

这一层重点回答：

**在结果不变的前提下，能不能更快找到路。**

---

### 5.3 第三层：方向状态扩展的带权 A* 多目标优化

当我们将转弯代价、风险代价和狭窄区域代价纳入模型后，路径规划问题就从“最短步数”升级为“最小综合代价”。

此时：

* BFS 已不再适用；
* 普通 A* 需要扩展为带权 A*；
* 节点状态需从位置 \(v\) 扩展为 \((v,d)\)，以显式处理方向变化。

改进后的评价函数可以写为：

\[
f(n)=g(n)+h(n)+\lambda_1 T(n)+\lambda_2 R(n)+\lambda_3 Q(n)
\]

其中：

* \(g(n)\) 表示累计实际代价；
* \(h(n)\) 表示到目标的估计代价；
* \(T(n)\) 表示转弯惩罚；
* \(R(n)\) 表示风险惩罚；
* \(Q(n)\) 表示狭窄区域或拥堵惩罚。

这一层体现的是**目标函数优化**，也是全项目最能体现运筹学特色的一层。

这一层重点回答：

**不只追求短，还要追求稳、顺、风险低。**

---

### 5.4 第四层：LPA* 与 D* Lite 的动态重规划优化

在迷宫竞赛实际环境中，地图并不一定始终静态。机器人运行中可能发现：

* 某条原本可行的通道被临时封闭；
* 某片区域新增障碍；
* 原有最优路径在当前时刻已不再可行。

此时需要快速重规划。

本项目在动态重规划层同时考虑以下两种增量式算法：

1. **LPA\***
   更偏理论化，强调在已有搜索结果基础上进行增量更新，适合展示动态最短路的状态继承思想；
2. **D* Lite**
   更偏机器人动态环境应用，适合解释“边走边修正路径”的过程，强调实时性与实用性。

与“每次从头重新运行 A*”相比，这类增量式算法能够更好利用既有信息，从而减少重复计算开销。将两者同时纳入项目，还可以进一步比较不同动态算法在重规划时间、稳定性和实现复杂度方面的差异。

这一层强调算法的**在线适应能力**，对应的是“从静态优化走向动态优化”。

这一层重点回答：

**环境变了以后，能不能快速改路。**

结合目前小组成员的实际想法，更适合采用“基础层 1 人、优化层 1 人、动态层 2 人”的分工方式。这样既能保持主线清晰，也能让工作量更加均衡。

| 成员 | 负责模块 | 核心内容 | 主要输出 |
| ---- | -------- | -------- | -------- |
| Hibiscus | BFS 与基础 A* | 迷宫地图建模、BFS 基准求解、标准 A* 实现，以及两者在最短路与搜索效率上的对比 | 基准实验结果、BFS 与 A* 对比分析 |
| 混合2402何 | A* 优化层 | 方向状态扩展、转弯惩罚、风险代价设计，以及带权 A* 或改进 A* 的实现与分析 | 多目标优化模型、改进 A* 实验结果 |
| Evan | LPA* 动态重规划 | 动态障碍更新下的增量式最短路求解，重点分析状态继承与重规划效率 | LPA* 理论与实验分析 |
| 成员 D | D* Lite 动态重规划 | 面向机器人在线运行过程的动态路径修正，重点分析实时性与路径更新效果 | D* Lite 理论与实验分析 |

从工作量上看，这样的划分是基本合理的：

1. BFS 本身难度较低，因此与基础 A* 放在同一位成员处，整体负担仍然可控；
2. A* 优化层需要负责方向状态扩展与综合代价建模，理论和实现深度较强，单独分配较为合理；
3. LPA* 与 D* Lite 虽同属动态重规划算法，但各自原理、更新机制和实验重点不同，分别由两位成员负责，工作量较为均衡；
4. 汇报时可形成“基准层 -> 优化层 -> 动态层双算法对照”的完整结构，不会显得任务割裂。

若后续需要进一步平衡工作量，可以默认由 Hibiscus 额外承担实验总表的整理，或由两位动态算法负责人共同完成动态场景数据汇总。

## 七、统一接口与实验设计

### 7.1 地图数据来源

为保证实验地图具有真实性与可复现性，建议优先采用历年 Micromouse 竞赛公开迷宫作为数据来源。具体可按以下原则处理：

1. 以历年公开迷宫作为原始样本，保证地图来自真实竞赛环境；
2. 以标准 `16 x 16` 迷宫为主，便于与竞赛规则保持一致；
3. 从同一来源中选取若干代表性地图，覆盖“通道较直”“转角较多”“死胡同较多”等不同结构特点；
4. 对动态实验，不额外更换底图，而是在同一标准地图上人为加入临时障碍或封路信息。

这样既能说明数据不是凭空构造，也便于四位成员在同一基础上复现实验。

---

### 7.2 统一地图输入格式

为了避免每个人使用不同地图表示方法，建议全组统一地图接口。每张地图至少包含以下字段：

| 字段 | 含义 |
| ---- | ---- |
| `maze_id` | 地图编号，便于统一记录 |
| `width` | 地图宽度，默认 16 |
| `height` | 地图高度，默认 16 |
| `wall_matrix` | 墙体信息或邻接阻塞信息 |
| `start` | 起点坐标 |
| `goal` | 终点坐标 |
| `risk_cells` | 风险区域集合，可为空 |
| `narrow_cells` | 狭窄区域集合，可为空 |
| `dynamic_updates` | 动态障碍更新时间与位置，可为空 |

其中：

1. `BFS` 与基础 `A*` 只需要使用 `wall_matrix`、`start`、`goal`；
2. 改进 `A*` 额外读取 `risk_cells` 与 `narrow_cells`；
3. `LPA*` 与 `D* Lite` 额外读取 `dynamic_updates`。

这样可以保证四种算法共享同一批地图文件，只是在读取字段时各有侧重。

---

### 7.3 统一算法输出接口

建议所有算法运行后统一输出如下结果，便于后续直接汇总成总表：

| 字段 | 含义 |
| ---- | ---- |
| `maze_id` | 当前测试地图编号 |
| `algorithm` | 算法名称 |
| `success` | 是否成功找到可行路径 |
| `path` | 最终路径坐标序列 |
| `path_length` | 路径长度 |
| `turn_count` | 转弯次数 |
| `explored_nodes` | 搜索或扩展节点数 |
| `runtime_ms` | 运行时间 |
| `total_cost` | 综合代价，静态基础算法可留空或等于路径长度 |

对于动态算法，再额外统一输出：

| 字段 | 含义 |
| ---- | ---- |
| `replan_count` | 重规划次数 |
| `replan_time_ms` | 单次或平均重规划时间 |
| `updated_nodes` | 动态更新中受影响的节点数 |
| `replanned_path_length` | 更新后路径长度 |

这样一来，你们最后只要把各自算法的结果整理到统一表头下，就可以直接横向比较。

---

### 7.4 实验场景设计

建议统一使用以下三类场景：

1. **基础通关场景**
   地图静态、通道代价一致，只考虑最短到达；
2. **高风险绕行场景**
   部分区域靠近障碍或通道狭窄，需要在距离与安全性之间权衡；
3. **突发封路场景**
   机器人运行过程中发现新障碍，需要重新规划路径。

如果老师或答辩同学追问技术细节，再说明这三类场景在实现上分别对应无权网格、带权网格和动态网格。

---

### 7.5 统一评判标准

建议统一采用以下指标：

| 指标 | 含义 |
| ---- | ---- |
| 路径长度 | 是否缩短移动距离 |
| 转弯次数 | 是否提高运行平稳性 |
| 综合代价 | 是否兼顾距离、转弯和风险 |
| 搜索节点数 | 是否减少无效扩展 |
| 运行时间 | 是否提升计算效率 |
| 重规划时间 | 环境变化后恢复能力如何 |
| 成功率 | 在不同场景下是否稳定找到可行路径 |

其中可以进一步约定：

1. `BFS` 与基础 `A*` 重点比较路径长度、搜索节点数和运行时间；
2. 改进 `A*` 重点比较综合代价、转弯次数和风险规避能力；
3. `LPA*` 与 `D* Lite` 重点比较重规划时间、更新节点数和路径恢复效果。

---

### 7.6 对比关系设计

为了体现递进性，实验不建议做成“所有算法彼此乱比”，而建议按层比较：

1. BFS vs A*
   比较在相同目标下谁更高效；
2. A* vs 带权 A*
   比较在加入现实代价后谁更合理；
3. LPA* vs D* Lite
   比较在地图更新后谁的重规划响应更快、路径调整更稳定。

这样实验逻辑会更清晰，结论也更容易写。

---

## 八、预期结果

预期可以得到如下结论：

1. 在无权静态场景中，BFS 能稳定求得最短路径，但搜索范围较大；
2. A* 能在不改变最优性的前提下减少搜索节点数，提高求解效率；
3. 当加入转弯和风险代价后，带权模型比单纯最短距离模型更符合竞赛机器人实际需求；
4. 方向状态扩展的带权 A* 能够更合理地求得综合代价最小路径；
5. LPA* 与 D* Lite 都能在环境变化时快速给出新路径，但两者在更新效率和应用侧重点上会表现出差异。

---

## 九、项目特色与创新点

本项目的特色与创新点建议总结为以下四点：

1. **情境具体，不再停留于抽象迷宫**
   将问题包装为迷宫竞赛机器人路径规划，使研究对象更真实、更贴近大学生竞赛语境，也更方便展示。

2. **算法结构具有明显递进关系**
   从静态无权最短路、搜索效率优化、多目标带权优化，到动态重规划，形成连续的优化链条。

3. **引入方向状态扩展建模**
   将“位置 + 方向”作为状态，正式刻画转弯惩罚，增强模型深度与运筹学特色。

4. **兼顾静态优化与动态适应**
   不只求一条最短路，还考虑环境更新后的快速重规划，更符合机器人竞赛中的真实需求。

## 十、总结

经过调整后，本项目不再是“几个常见迷宫算法的简单比较”，而是一个围绕具体竞赛场景展开的路径优化研究：先解决静态无权最短路，再优化搜索效率，随后引入多因素综合代价，最后处理动态环境下的快速重规划问题。

这样的设计有三个明显优势：

1. 有清晰场景，汇报时更容易讲故事；
2. 有明确主线，算法之间不是并列堆砌，而是层层升级；
3. 有运筹学味道，不仅有最短路，还有带权优化、多目标建模与动态决策。
