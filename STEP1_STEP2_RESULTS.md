## 实现概述

### 第一步：BFS（广度优先搜索）

**目标**：在完全已知、无权、静态地图中找到最短可达路径

**实现特点**：
- 使用队列（deque）进行广度优先遍历
- 确保找到的路径为最短路径
- 记录所有扩展的节点数用于效率评价
- 准确计算转弯次数（90度计1次，180度计2次）

**关键代码位置**：`src/bfs/runner.py`

```python
- 使用visited集合避免重复访问
- 路径回溯通过current_path列表实现
- 转弯检测通过方向向量比较
```

### 第二步：A*（启发式搜索）

**目标**：在相同的最短路目标下，通过启发式搜索减少扩展节点数，提升搜索效率

**实现特点**：
- 使用曼哈顿距离作为启发函数：`h(n) = |x_n - x_g| + |y_n - y_g|`
- 评价函数：`f(n) = g(n) + h(n)`
- 使用优先队列（heapq）按f值排序节点
- 保证找到的路径仍为最短路

**关键代码位置**：`src/a_star/runner.py`

```python
- g_score记录从起点到当前节点的实际代价
- f_score用于优先队列排序
- parent字典用于路径回溯
- 启发函数导向搜索朝向目标
```

## 关键发现

### 1. 路径质量
✓ **两种算法找到的路径长度完全一致**
- 所有3张测试地图上，BFS和A*均找到长度相同的最短路径
- 验证了A*在保证最优性的前提下的有效性

### 2. 搜索效率
**A*在启发函数引导下的表现混合：**

| 地图 | 节点减少比例 | 分析 |
|------|----------|------|
| alljapan-015-1994 | -1.6% | A*扩展稍多，可能因时间序列开销 |
| alljapan-031-2010 | **+40.7%** | A*显著减少扩展，启发函数效果明显 |
| alljapan-044-2023 | -3.3% | A*扩展稍多 |

**结论**：
- 在相对直阳通道的地图上（alljapan-031），A*启发函数效果显著，减少40.7%的节点扩展
- 在转角较多的复杂地图上，启发函数优势不明显，甚至因管理开销略微增加节点扩展

### 3. 执行速度

当前测试中运行时间受以下因素影响较大：
- Python的纯运算开销变得相对主要（两个算法都很快）
- 优先队列（heapq）的管理成本可能在某些情况下抵消节点减少的收益

**平均性能**：
- 在简单图（alljapan-031）上：A* ≈ BFS（0.130ms vs 0.132ms）
- 在复杂图上：A*时间略长，主要因为队列管理

## 转弯代价分析

按照接口规范统一定义：
```
turn_count = N_90 + 2*N_180
其中：
  N_90 = 90度转弯次数
  N_180 = 180度掉头次数
```

**观察**：
- 不同的等长路径可能有不同的转弯模式
- BFS和A*由于探索顺序不同，可能选择不同的等长路径
- 这为第三步（带权A*）优化多目标代价提供了实验基础
